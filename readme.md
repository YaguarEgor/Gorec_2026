# Горец-2025
## Документация Telegram-бота игры «Горец»

Переработанная версия бота для игры **«Горец»**.  
Бот написан на **Python** с использованием:

- **aiogram** — Telegram API
- **SQLAlchemy (async)** — работа с базой данных
- **SQLite / PostgreSQL** — хранение данных
- **QR-кодов** — подтверждение убийств
- **локального хранения фото** — подтверждение убийств

Бот поддерживает **два режима игры**:

1. **Классический режим** — игрок охотится на одного конкретного игрока
2. **Командный режим** — команды охотятся на команды

---

# Структура проекта

## `main.py`
Главный файл Telegram-бота.

Здесь находится:

- создание `Bot` и `Dispatcher`
- подключение базы данных
- описание всех **команд и хендлеров**
- FSM-сценарии
- уведомления игрокам
- админские инструменты

### Основные функции и логика

#### Общие команды
- `/start` — начать работу с ботом
- `/help` — показать список команд
- `/cancel` — отменить текущее FSM-действие

#### Регистрация игроков
FSM регистрации включает:

1. выбор режима (`classic` или `team`)
2. ввод ФИО
3. ввод курса
4. загрузку фотографии лица
5. подтверждение заявки

Для **team-режима** дополнительно:
- ввод названия команды
- загрузка фотографии команды

#### Модерация заявок
Администратор может:

- одобрить заявку
- отклонить заявку
- указать причину отказа

Игрок получает уведомление о результате.

#### Классический режим

Команды:

- `/start_classic_round` — запуск раунда
- `/target` — показать текущую цель игрока

Логика:

- игроки образуют **цикл целей**
- после убийства цель переназначается
- когда остаётся один игрок — игра заканчивается

#### Командный режим

Команды:

- `/start_team_round` — запуск раунда
- `/team_target` — показать цель команды

Логика:

- команды образуют **цикл охоты**
- игрок может устранить **любого живого игрока команды-цели**
- после уничтожения команды назначается новая цель

#### QR-коды

Команда:


/my_qr


Игрок получает личный QR-код.

QR используется для подтверждения убийства.

#### Подтверждение убийства

Команда:



/kill



FSM сценарий:

1. ввод QR-кода
2. отправка фото
3. проверка в базе
4. запись убийства
5. уведомления игрокам

# Логика игры

## Классический режим

Игроки образуют цикл:

 
A → B → C → ... → A
 

После убийства:

* жертва выбывает
* убийца получает новую цель
* игра продолжается

Если остаётся **1 игрок** — он побеждает.

---

## Командный режим

Команды образуют цикл:

 
Team A → Team B → Team C → ... → Team A
 

Игрок может устранить **любого игрока команды-цели**.

Если команда полностью уничтожена:

* назначается новая цель
* начисляются очки

---

# Подтверждение убийства

Убийство подтверждается:

1. командой `/kill`
2. вводом QR-кода
3. загрузкой фото
4. проверкой данных
5. записью в таблицу `kills`
